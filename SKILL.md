# Course Review Skill — 基于费曼学习法的课程速通复习方案生成器

> **核心理念**：用费曼学习法，帮基础薄弱的学生在最短时间内建立课程框架、抓住考试重点。
> **不是固定模板**：AI应根据课程特点自主增减模块，本skill提供的是设计思路和参考实现，而非刻板框架。

---

## 给AI的阅读指南（重要）

如果你是AI，正在阅读这个skill来为用户创建复习方案，请遵循以下原则：

### 第一原则：费曼学习法是灵魂

你创建的每一个复习页面，都必须贯彻费曼学习法的四步精神：

| 费曼步骤 | 在复习页中的体现 | 你必须做到 |
|----------|-----------------|-----------|
| ① 用最简单的话解释 | 每个概念卡片的"白话版" | 用日常语言重新描述，禁止术语堆砌 |
| ② 用类比建立直觉 | 每个核心概念配类比 | 找一个学生熟悉的场景/模型做类比 |
| ③ 主动暴露盲区 | "常见误区" + 自测清单 | 写出学生最容易犯的错，而非只写正确答案 |
| ④ 压缩为速记口诀 | 速记口诀区 | 用最少的字把核心框架记住 |

### 第二原则：灵活组合，而非照搬模板

**不要照搬任何已有复习页的全部模块。** 你应该分析课程特点，决定：
- **哪些模块需要？** 概念型课程不需要公式速查；文科课程不需要计算题库
- **模块顺序如何排？** 编程课可能先放代码示例再放概念；历史课可能先放时间线
- **内容深度多深？** 完全零基础的学生需要更多白话和类比；有一定基础的可以更精炼
- **需要哪些题型？** 看考试形式——纯选择题的考试不需要简答题库

### 第三原则：知识关系图是核心

知识关系图不是简单的"第1章→第2章→第3章"线性罗列，而是要展示**知识点之间的逻辑关系**。详见下文"知识关系图设计指南"。

### 第四原则：面向基础薄弱+短时间

你的读者是：
- 对课程整体框架缺乏认知的人
- 核心概念理解不牢固的人
- 只有1-3天复习时间的人
- 需要直观、通俗解释而非学术语言的人

**每一个设计决策都要问自己：基础薄弱的学生看得懂吗？短时间能看完吗？**

### 第五原则：课程类型决定设计哲学（v2.1新增）

**不同类型的课程需要不同的"降维"策略。** 费曼白话法（语言降维）适合概念型/文科课程，但对计算型/空间想象型课程可能失效甚至误导。AI必须先判断课程类型，再决定采用哪种设计哲学。详见下文"计算型课程差异化设计指南"。

### 第六原则：参考材料优先于AI推测（v2.2新增·最重要）

**AI生成的知识点和题目基于训练数据，可能与用户学校的实际考试内容不一致。** 不同学校的考点侧重、出题风格、难度偏好差异很大，纯靠AI推测容易"复习偏了"。

**因此，AI在生成复习方案前，必须主动询问用户是否有以下参考材料：**

| 参考材料类型 | 价值 | 如何使用 |
|-------------|------|---------|
| 往年试卷/期末卷 | **最高价值**——直接反映出题风格、考点分布、难度水平 | 提取原题加入题库；分析考点权重调整章节优先级；模仿出题风格补充相似题 |
| 平时作业/练习题 | 高价值——反映老师关注的重点和常见题型 | 提取典型题加入题库；识别高频考点 |
| 上课课件/PPT | 中高价值——反映教学大纲和老师强调的重点 | 校准知识点覆盖范围；识别老师额外强调的内容 |
| 教学大纲/考试范围 | 高价值——明确考什么、不考什么 | 精准划定复习边界，删除不考的内容 |
| 教材课后习题 | 中价值——基础题型参考 | 补充基础练习题 |

**核心原则：有参考材料时，以参考材料为准；无参考材料时，AI应明确告知用户内容基于推测，并建议用户自行校验。**

详见下文"参考材料使用指南"。

### 第七原则：例题驱动优于概念罗列（v2.3新增）

对高数、物理等计算型/应用型课程，学生最大的痛点是"概念都懂但不会做题"。传统的"先讲概念再看例题"顺序效果不好——概念太抽象，看完就忘，不知道什么时候用。

**例题驱动反过来：先给一道典型题，从解题过程中学概念。** 学生学完一道题，既会了做题方法，又理解了背后的概念，还能通过变式练习检验是否真的掌握。

| 传统模式（概念驱动） | 例题驱动模式（v2.3） |
|---------------------|---------------------|
| 概念1→概念2→概念3→例题 | 例题→思路引导→分步解→知识提炼→变式练习 |
| 看完概念不会做题 | 学完一题会一类题 |
| 适合概念型文科课程 | 适合计算型/应用型课程 |

**关键原则**：例题驱动的核心不是"多做题"，而是"从一道题里学透一类题"。每道例题必须包含「怎么想到的」（思路引导）和「变式练习」（迁移检验）。

详见下文"例题驱动设计指南"。

## 指令

### 步骤一：收集参考材料（v2.2新增·必须执行）

**这是生成复习方案前的第一个步骤，不可跳过。** AI必须主动向用户询问参考材料。

#### 1.1 询问话术模板

AI应在收到用户的复习请求后，**立即**询问以下内容（可根据上下文调整措辞）：

```
在开始生成复习方案之前，我想先了解一下你手上是否有以下参考材料。
这些材料能帮我更精准地匹配你们学校的考点和出题风格，避免"复习偏了"：

1. 📝 往年试卷/期末卷（最有价值——直接反映考点和难度）
2. 📋 平时作业/练习题（反映老师关注的重点）
3. 📊 上课课件/PPT（反映教学大纲和重点）
4. 📖 教学大纲/考试范围（明确考什么、不考什么）
5. 📚 教材课后习题（基础题型参考）

你可以提供以下任意形式：
- 直接上传文件（PDF、Word、图片、PPT等）
- 粘贴文字内容
- 描述考点和题型（如"往年考了3道Green公式大题"）

即使没有这些材料也没关系，我会基于通用知识生成方案，
但会在页面中标注哪些内容是基于推测的，建议你自行校验。
```

#### 1.2 根据用户回应采取不同策略

| 用户回应 | AI的行动 |
|---------|---------|
| 提供了往年试卷 | **最高优先级**：提取原题加入题库，分析考点分布调整章节权重，模仿出题风格补充相似题。在页面中标注"📌 来源于往年试卷" |
| 提供了平时作业/练习题 | 提取典型题加入题库，识别高频考点。在页面中标注"📌 来源于平时作业" |
| 提供了上课课件/PPT | 校准知识点覆盖范围，识别老师额外强调的内容。在页面中标注"📌 课件强调" |
| 提供了教学大纲/考试范围 | 精准划定复习边界，删除不考的章节。在页面中标注"📌 考试范围" |
| 提供了教材课后习题 | 补充基础练习题 |
| 描述了考点/题型（无文件） | 根据描述调整知识点权重和题型分布 |
| 表示没有参考材料 | 明确告知用户：内容基于AI推测，在页面显著位置添加提醒"⚠️ 本方案基于通用知识生成，考点可能与你校实际考试有偏差，建议结合老师强调的重点自行校验" |

#### 1.3 参考材料处理原则

1. **原题优先**：往年试卷中的原题直接加入题库，不做改写（除非有版权顾虑）
2. **风格模仿**：分析往年试题的出题风格（侧重计算/侧重概念/侧重应用），补充相似风格的题目
3. **权重校准**：根据参考材料中各知识点出现的频率，调整章节优先级和知识点权重
4. **边界划定**：教学大纲/考试范围明确不考的内容，从复习方案中删除或标注"不考"
5. **透明标注**：所有来源于参考材料的内容，在页面中标注来源，让用户知道哪些是"真题风格"、哪些是"AI推测"

### 步骤二：分析课程信息

向用户收集以下信息（如用户未提供）：

1. 课程名称
2. 课程类型（概念型/计算型/实践型/混合型）
3. 章节范围
4. 考试形式（选择题/简答题/计算题/论文/混合）
5. 学生基础（完全零基础/有一定基础）
6. 复习时间（1天/2天/3天）
7. 核心概念（3-5个最重要的概念）

### 步骤三：确定模块组合

根据课程类型选择模块：

**必选模块（任何课程都必须包含）：**

| 模块 | 费曼法对应 | 为什么必选 |
|------|-----------|-----------|
| 新手引导区 | 整体认知 | 基础薄弱学生的第一入口 |
| 知识关系图 | 理解关系 | 用关系代替背诵 |
| 概念精讲区 | 白话+类比 | 费曼法核心实践 |
| 自测清单 | 暴露盲区 | 主动发现"我不会什么" |

**可选模块（按需选用，不要全部加上）：**

| 模块 | 适用课程 | 不适用课程 |
|------|---------|-----------|
| 悬浮解释框 | 有专业术语的课程 | 概念极少的课程 |
| 概念搜索面板 | 概念多、需快速定位 | 概念少（5个以内） |
| 复习仪表盘 | 内容多、需进度管理 | 内容少（3章以内） |
| 公式速查 | 理科、计算类 | 纯文科、概念类 |
| 概念图表化 | 有图形/模型/流程 | 纯文字概念 |
| 应试急救包 | 有固定题型 | 开卷考、论文考 |
| 题库系统 | 需大量练习 | 以理解为主 |
| 章节优先级筛选 | 章节多需取舍 | 章节少 |
| 代码示例 | 编程类 | 非编程类 |
| 时间线/历史脉络 | 历史、法律 | 数学、编程 |
| 对比表格 | 有大量对比 | 概念独立 |
| 考前一页纸 | 任何课程 | — |
| **知识盲区诊断**（v2.1新增） | 任何课程 | — |
| **计算依赖树**（v2.1新增） | 计算链式递进课程 | 概念独立 |
| **统一性主线表**（v2.1新增） | 有统一性主线的课程 | 概念独立 |
| **方法决策树**（v2.1新增） | 有多种方法选择 | 只有一种方法 |
| **交互式几何可视化**（v2.1新增） | 依赖空间想象 | 纯文字概念 |
| **分步计算填空**（v2.1新增） | 计算型课程 | 纯概念型 |
| **计算陷阱库**（v2.1新增） | 计算型课程 | 纯概念型 |

**决策流程：**

```
分析课程 → 确定课程类型 → 判断是否需要差异化设计 → 选择模块 → 决定顺序 → 填充内容
    │
    ├── 概念型课程（如哲学、文学史）
    │   → 新手引导 + 知识关系图(总分/对比) + 概念精讲 + 时间线 + 自测清单
    │
    ├── 计算型课程（如高数、物理、线代）──→ 见"计算型课程差异化设计指南"
    │   → 新手引导 + 诊断测试 + 计算依赖树 + 统一性主线表 + 方法决策树
    │     + 例题驱动精讲(v2.3推荐) + 几何可视化 + 分步计算填空 + 计算陷阱库
    │     + 题库 + 自测清单 + 考前一页纸
    │
    ├── 实践型课程（如编程）
    │   → 新手引导 + 知识关系图(前提依赖) + 例题驱动精讲 + 代码示例 + 题库 + 自测清单
    │
    └── 混合型课程（如经济学）
        → 新手引导 + 知识关系图(因果/循环) + 概念精讲 + 概念图表化 + 公式速查 + 应试急救包 + 题库 + 自测清单
```

## 参考材料使用指南（v2.2核心新增）

> **背景**：AI生成的知识点和题目基于训练数据，可能与用户学校的实际考试内容不一致。本章节定义了如何利用用户提供的参考材料来校准复习方案。

### 为什么需要参考材料？

不同学校、不同老师的考试存在显著差异：

| 差异维度 | 示例 | 仅靠AI推测的风险 |
|---------|------|-----------------|
| 考点侧重 | 同是高数下册，A校重点考Green公式，B校重点考级数 | 复习了Green公式结果考级数——白费功夫 |
| 出题风格 | A校偏概念理解，B校偏复杂计算 | 练了一堆计算题结果考概念——方向错误 |
| 难度水平 | A校期末简单，B校期末接近考研难度 | 难度不匹配——要么复习过度焦虑，要么复习不足挂科 |
| 题型分布 | A校只有大题，B校选择填空计算各占1/3 | 题型练错了——考试时手生 |
| 特殊偏好 | 某老师喜欢考"证明题"或"应用题" | 漏练了关键题型 |

### 各类参考材料的使用方法

#### 1. 往年试卷/期末卷（最高价值）

**处理流程**：
```
接收试卷 → 提取所有题目 → 分类（按题型/按章节）
    ↓
分析考点分布 → 计算各章节权重 → 调整复习方案的章节优先级
    ↓
分析出题风格 → 判断侧重（概念/计算/应用）→ 补充相似风格的练习题
    ↓
原题入题库 → 标注"📌 来源于往年试卷" → 模仿出题风格生成相似题
```

**题库标注示例**：
```javascript
{
  question: '计算曲线积分 ∮(xdy - ydx)，其中L是x²+y²=R²的逆时针方向',
  explanation: '...',
  source: '往年试卷',  // v2.2新增：标注来源
  year: '2023期末',   // v2.2新增：标注年份（如有）
  chapter: 5,
  score: 15
}
```

#### 2. 平时作业/练习题

- 提取典型题目加入题库，标注"📌 来源于平时作业"
- 识别高频出现的知识点（老师反复布置的作业通常是重点）
- 识别老师偏好的题型

#### 3. 上课课件/PPT

- 校准知识点覆盖范围：课件中有的内容纳入复习，课件中没有的可降低优先级
- 识别老师额外强调的内容（课件中用红色/加粗/★标注的内容）
- 在概念精讲区标注"📌 课件强调"标签

**HTML标注示例**：
```html
<div class="concept-card">
  <h3>Green公式 <span class="exam-source-badge">📌 往年高频考点</span></h3>
  <!-- ... -->
</div>
```

#### 4. 教学大纲/考试范围

- 明确不考的章节：从复习方案中删除，或保留但标注"⚠️ 大纲标注不考，仅供参考"
- 明确重点章节：在章节优先级中提升权重
- 明确考试形式：调整题库的题型分布

#### 5. 教材课后习题

- 补充基础练习题，标注"📌 教材习题"
- 适合作为入门级练习


### 参考材料与费曼学习法的关系

参考材料不影响费曼学习法的"解释方式"（白话/类比/误区），而是影响"解释什么"：

| 维度 | 费曼学习法负责 | 参考材料负责 |
|------|--------------|-------------|
| 概念解释方式 | ✅ 白话版、类比、误区 | ❌ 不涉及 |
| 知识点权重 | ❌ 不涉及 | ✅ 哪些考点更重要 |
| 题目内容 | ❌ 不涉及 | ✅ 练什么题、什么风格 |
| 章节优先级 | ❌ 不涉及 | ✅ 哪些章节重点复习 |
| 考试范围 | ❌ 不涉及 | ✅ 考什么、不考什么 |

**简单记忆**：费曼法管"怎么讲"，参考材料管"讲什么、练什么"。

## 计算型/空间想象型课程差异化设计指南（v2.1核心新增）

> **背景**：费曼白话法（语言降维）适合概念型/文科课程，但对高数、物理、线代等强计算+强空间想象的课程，强制"生活类比"会失效甚至误导。本章节定义了这类课程的差异化设计逻辑。

### 何时启用差异化设计

当课程具备以下**任一特征**时，应在费曼白话法基础上叠加（或部分替代）差异化设计：

| 特征 | 示例课程 | 差异化设计的价值 |
|------|---------|----------------|
| 大量数学公式和符号运算 | 高数、线代、概率论 | 纯文本无法表达公式，必须KaTeX渲染 |
| 强依赖空间想象和几何直觉 | 解析几何、电磁学、力学 | 生活类比无法建立空间直觉，必须可视化 |
| 知识点是计算链式递进 | 高数（偏导→重积分→三大公式） | 前一步算不对后面全错，需要阻塞预警 |
| 存在"统一性主线" | 高数（N-L→Green→Gauss→Stokes） | 找到"纲"可以一统多，而非按章死记 |
| 有多种方法/坐标系选择 | 高数（直角/极/柱/球面坐标） | 选对方法就成功一半，需要决策引导 |

### 费曼四步法的适用性调整

计算型课程中，费曼四步法的映射关系需要调整：

| 费曼步骤 | 概念型课程（标准费曼） | 计算型课程（差异化） |
|----------|---------------------|---------------------|
| ① 用最简单的话解释 | 白话版（日常语言） | **几何直观优先**：用图形/动画建立直觉，辅以白话 |
| ② 用类比建立直觉 | 生活场景类比（吃饭、排队） | **数学类比**：低维→高维推广、特殊→一般 |
| ③ 主动暴露盲区 | 概念理解误区 | **计算操作陷阱**：忘乘r、方向弄反、未检查封闭等 |
| ④ 压缩为口诀 | 速记口诀 | 速记口诀 + **统一性主线**：找课程"纲" |

**关键原则**：
- 不强制每个概念配生活类比——对抽象数学概念，强行找生活类比是牵强甚至误导的
- 用"几何直观"替代"白话解释"作为第一直觉建立方式
- 用"低维→高维推广"替代"生活场景类比"
- 一张能动的图胜过一千字白话

### 六大设计原则

#### 原则1：公式渲染基础设施

计算型课程的公式必须用 KaTeX/MathJax 渲染，**禁用纯文本写数学公式**。这是基础设施，不可省略。

```html
<!-- 在head中引入KaTeX -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/katex.min.css">
<script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/katex.min.js"></script>
<script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/contrib/auto-render.min.js"></script>

<!-- 渲染配置 -->
<script>
function renderMath(){
  if(typeof renderMathInElement !== 'undefined'){
    renderMathInElement(document.body,{
      delimiters:[
        {left:'$$',right:'$$',display:true},
        {left:'$',right:'$',display:false}
      ],
      throwOnError:false,
      strict:false  // 容忍中文混入数学模式
    });
  }
}
</script>
```

#### 原则2：几何直观优先

**用图形和动画建立直觉，替代白话解释。** 对高数而言：
- 偏导数 → "站在山坡上，只看东西方向的坡度"（几何：切线斜率）
- 梯度 → "指向最陡上坡方向的箭头"（几何：法向量）
- 二重积分 → "曲顶柱体的体积"（几何：3D分割图）

**类比方式的转变**：
- 用"低维→高维推广"：定积分（面积）→ 二重积分（体积）→ 三重积分（超体积）
- 用"特殊→一般"：Green公式是Stokes公式的二维特例
- 用"几何意义"：旋度=旋转程度，散度=源汇强度

#### 原则3：计算依赖链+阻塞预警

计算型课程的知识关系是**计算层面的递进链**，而非概念层面的关系。学生真正需要知道的是"如果你算不对偏导数，后面全错"这种**计算阻塞关系**。

**用计算依赖树替代概念关系图**：
- 展示计算链：偏导数 → 全微分 → 梯度 → Lagrange乘数法
- 标注红色"计算阻塞点"：偏导数算不对 → 重积分全错 → 三大公式全废
- 点击节点可查看详情和跳转

#### 原则4：找统一性主线

**这是计算型课程最核心的"纲"。** 识别贯穿全书的统一主线，用一张表让学生看到"原来这些公式本质相同"。

**示例（高数下册）**：
```
Newton-Leibniz（一维：边界值 = 区间上导数的积分）
    ↓ 推广到二维
Green公式（环路积分 = 区域内偏导的积分）
    ↓ 推广到三维
Gauss公式（闭曲面积分 = 体内散度的积分）
    ↓ Green的三维版
Stokes公式（边界环量 = 曲面旋度的积分）
```

让学生看到"四个公式是一回事"，而非四个孤立公式。这比费曼的"生活类比"更能建立深层理解。

#### 原则5：分步计算填空

**用交互式推导链替代静态题库。** 将关键计算流程做成分步填空：
```
题目 → 第1步(填空) → 校验 → 第2步(填空) → 校验 → ... → 最终结果
```
每步即时反馈对错，而非只看最终答案。这比静态展示解题步骤更有效。

#### 原则6：知识盲区诊断测试

**在页面开头先做核心诊断题，自动标记盲区并跳转学习。**

流程设计：
```
打开页面 → 做6道核心诊断题（每题关联1个知识点）
    ↓
答错或点"不会" → 自动标记为盲区
    ↓
生成盲区报告 → 显示评级 + 盲区列表
    ↓
点击"跳转学习" → 展开对应章节 + 滚动到概念精讲
```

这让复习更有针对性：先测出哪里不会，再精准补漏，而不是盲目从头看到尾。

### 计算型课程专属模块详解

| 模块 | 作用 | 实现要点 | 适用条件 |
|------|------|---------|---------|
| KaTeX公式渲染层 | 数学公式排版 | CDN引入，strict:false容错 | 有数学公式 |

| 分步计算填空 | 过程性练习 | 每步填空+即时校验+完整答案 | 计算型课程 |

### KaTeX使用避坑

1. **数学模式中禁止中文**：`\text{收敛}` 会触发警告，改为 `收敛 $R=...$`

## 例题驱动设计指南（v2.3核心新增）

> **背景**：v2.1定义了计算型课程的差异化设计（几何直观、计算依赖树、统一性主线），但概念精讲区的组织方式仍然是"按概念罗列卡片"。学生反馈"概念都懂但不会做题"——因为概念和题目是分离的。v2.3新增例题驱动模式，把概念精讲区从"概念卡片"改为"例题卡片"，以题目为核心展开知识点。

### 什么是例题驱动？

**例题驱动 = 以典型题目为核心，从解题过程中提炼知识点。**

传统概念驱动的问题：
- 概念太抽象，看完就忘
- 不知道概念"什么时候用、怎么用"
- 做题时想不到用哪个知识点
- 概念和题目是两张皮

例题驱动的优势：
- 从题目出发，学完就会做
- 知识点从解题过程中自然引出，更容易理解
- 变式练习检验是否真的掌握
- 一道题覆盖多个知识点，效率更高

### 例题驱动与费曼学习法的对应关系

例题驱动不是对费曼法的替代，而是**计算型/应用型课程的费曼法变体**：

| 费曼步骤 | 概念型课程 | 例题驱动（v2.3） |
|----------|-----------|-----------------|
| ① 用最简单的话解释 | 白话版 | 「怎么想到的」——用最朴素的思路引导，不讲术语 |
| ② 用类比建立直觉 | 生活类比 | 分步解题——每一步都是前一步的自然延伸，从已知到未知 |
| ③ 主动暴露盲区 | 常见误区 | 变式练习——换个条件还会不会？暴露假懂 |
| ④ 压缩为口诀 | 速记口诀 | 「从题目学到的」——把方法压缩为可复用的套路 |

### 例题卡片的标准结构（五段式）

每道例题必须包含以下五个部分，缺一不可：

```
┌─────────────────────────────────────┐
│  例题标题 + 来源/分值标签            │  ← example-header
├─────────────────────────────────────┤
│  📝 题目内容                         │  ← ex-problem
├─────────────────────────────────────┤
│  💡 怎么想到的？（思路引导）          │  ← ex-thinking
├─────────────────────────────────────┤
│  ① 第一步...                        │
│  ② 第二步...                        │  ← ex-steps（多步）
│  ③ 第三步...                        │
│     → 结果高亮                       │
├─────────────────────────────────────┤
│  📚 从题目学到的（知识提炼）          │  ← ex-knowledge
├─────────────────────────────────────┤
│  🔄 变式练习（点击展开）              │  ← ex-variant（可折叠）
└─────────────────────────────────────┘
```

### 各部分详细要求

#### 1. 题目展示（ex-problem）

- **优先选用真题**：往年试卷、模拟卷中的原题
- **标注来源和分值**：让学生知道这道题的重要性
- **公式用KaTeX渲染**：保持与正文一致的阅读体验
- 题目要完整，条件要清晰

#### 2. 怎么想到的（ex-thinking）

这是例题驱动的**灵魂部分**——不能直接给答案，要讲清楚"为什么想到这个方法"。

**好的思路引导**：
> 积分区域是圆 + 被积函数含 $x^2+y^2$ → 极坐标！别忘了乘 $r$。

**差的思路引导**（等于没说）：
> 用极坐标计算。

**引导的角度**：
- 从**题目特征**出发：看到什么信号想到用什么方法
- 从**排除法**出发：为什么不用直角坐标？为什么不用换元法？
- 从**目标倒推**：要求的结果是什么？需要什么条件才能得到？

#### 3. 分步解题（ex-steps）

- **编号步骤**：用①②③或1️⃣2️⃣3️⃣，每步一个明确的操作
- **每步有输入输出**：这一步做了什么？得到了什么结果？
- **关键结果高亮**：每一步的结果用绿色背景框标出
- **步骤数控制**：3-6步为宜，太少没过程感，太琐碎学生记不住

**步骤设计原则**：
- 从已知到未知，每一步都是前一步的自然延伸
- 关键变形处要有说明（如"换元法：令u=-r²"）
- 不要跳步——基础薄弱的学生会卡在你跳过的那一步

#### 4. 从题目学到的（ex-knowledge）

这是**知识提炼**部分——从一道具体的题，抽象出可复用的方法和知识点。

**提炼内容包括**：
- **知识点**：这道题涉及哪些核心概念？
- **方法套路**：这类题的通用解法是什么？
- **判断信号**：看到什么特征就知道用这个方法？
- **易错提醒**：这类题最容易在哪里出错？

**好的知识提炼**：
> ① 看到 $x^2+y^2$ 用极坐标——这是最重要的信号
> ② 极坐标 $d\sigma=r\,dr\,d\theta$，别忘了乘 $r$！（最高频错误）
> ③ $e^{-r^2}$ 积分用换元法，令 $u=-r^2$

#### 5. 变式练习（ex-variant）

这是**检验真懂假懂**的部分——改变条件，看学生还会不会做。

**变式的四个层次**：

| 层次 | 变式方式 | 检验什么 | 难度 |
|------|---------|---------|------|
| 1级 | 改变数字 | 是不是自己算的 | ⭐ |
| 2级 | 改变条件 | 方法有没有真掌握 | ⭐⭐ |
| 3级 | 反向提问 | 理解深度够不够 | ⭐⭐⭐ |
| 4级 | 综合提升 | 知识迁移能力 | ⭐⭐⭐⭐ |

**变式练习的设计原则**：
- 默认折叠，点击展开（避免给学生压力）
- 变式不要太多，1-2个就够
- 给出答案和简要提示，不要长篇大论
- 变式的目的是检验，不是再讲一道新题

#### 6. 配套小测（ex-practice，v2.4新增）

这是**巩固记忆**的部分——学完一道例题，马上做2-3道相关练习题，强化知识点的留存。

**配套小测与变式练习的区别**：

| 维度 | 变式练习 | 配套小测 |
|------|---------|---------|
| 位置 | 例题内部第五步 | 例题末尾独立模块 |
| 数量 | 1道（改变条件的同类题） | 2-3道（覆盖各知识点） |
| 目的 | 检验"这道题会不会" | 巩固"这些知识点懂没懂" |
| 难度 | 与例题接近 | 从基础到综合递进 |
| 答案 | 直接给出 | 每题独立折叠，做完一道看一道 |

**配套小测的设计原则**：

1. **紧扣例题知识点**：每道小题都要能从例题的知识提炼中找到对应
2. **2-3道为宜**：第1题基础（直接套用），第2题变形（换条件/换角度），第3题（可选）综合（结合其他知识点）
3. **答案独立折叠**：每题都有单独的"查看答案"按钮，避免一次看到所有答案
4. **详解不跳步**：答案要有完整解题过程，不能只给最终结果
5. **难度标注**：可用 ⭐ 标注难度，让学生心中有数

**配套小测的HTML结构**见下方模板示例。

### 选题原则

1. **真题优先**：从往年试卷/模拟卷中选，标注来源和分值
2. **每章2-3题**：覆盖本章核心考点，不要贪多
3. **从易到难**：先基础题后综合题
4. **一题多知识点**：一道题尽量覆盖多个知识点，提高效率
5. **选"母题"**：能衍生出多种变式的典型题，而不是偏题怪题

### 章节组织方式

例题驱动模式下，概念精讲区的组织方式：

```
第X章 · 章节名称（分值/重要性标签）
├── 本章核心：一句话概括本章考什么
├── 例题1：最基础/最高频的题型
│   └── 题目 → 思路 → 分步解 → 知识提炼 → 变式 → 配套小测
├── 例题2：次重要/稍难的题型
│   └── 题目 → 思路 → 分步解 → 知识提炼 → 变式 → 配套小测
└── （可选）例题3：综合题/难题
    └── 题目 → 思路 → 分步解 → 知识提炼 → 变式 → 配套小测
```

**与手风琴的配合**：章节标题仍是手风琴按钮，展开后里面是2-3张例题卡片，而不是概念卡片。

| 模块 | 与例题驱动的关系 |
|------|----------------|
| 知识盲区诊断 | 诊断发现盲区 → 跳转到对应章节的例题卡片 |

## 费曼学习法如何融入复习页

### 费曼法第一步：用最简单的话解释

**在概念精讲区，每个概念卡片必须包含"白话版"（概念型课程）或"几何直观"（计算型课程）：**

```html
<div class="concept-card">
  <h3>乘数效应</h3>
  <p><strong>定义：</strong>投资增加引起的国民收入增加量是投资增加量的倍数。</p>
  
  <!-- 概念型课程：白话版 -->
  <div class="plain-explain">
    <strong>白话版：</strong>政府花1块钱修路，修路工人拿到1块钱去买面包，
    面包师拿到钱去买衣服……这1块钱转了好几圈，最后GDP涨了不止1块钱。
    涨了几倍，这个"倍"就是乘数。
  </div>
</div>
```

**关键要求**：
- 禁止用另一个专业术语解释专业术语
- 概念型课程用日常语言；计算型课程用几何直观
- 宁可啰嗦也不要含糊

### 费曼法第二步：用类比建立直觉

**每个核心概念配一个类比：**
- **概念型课程**：生活场景类比（吃饭、排队、买东西）
- **计算型课程**：数学类比（低维→高维推广、特殊→一般）

```html
<div class="plain-explain">
  <strong>生活类比：</strong>
  IS-LM模型就像跷跷板——商品市场坐在一头，货币市场坐在另一头，
  利率是中间的支点。
</div>
```

**类比设计原则**：
- 概念型课程用学生一定熟悉的场景
- 计算型课程用数学推广关系（如"Green是Stokes的二维特例"）
- 类比要抓住概念的**核心机制**，而非表面相似

### 费曼法第三步：主动暴露盲区

**每个概念卡片包含"常见误区"（概念型）或"计算陷阱"（计算型）：**

```html
<!-- 概念型课程：概念理解误区 -->
<div class="common-mistake">
  <strong>常见误区：</strong>
  ① 很多人以为乘数越大越好——其实乘数大意味着经济波动更剧烈
</div>

<!-- 计算型课程：计算操作陷阱 -->
<div class="common-mistake">
  <strong>计算陷阱：</strong>
  ① 极坐标换元后忘记乘 r——这是最高频错误！
  ② 第二类曲线积分反向时忘记变号
</div>
```

### 费曼法第四步：压缩为速记口诀

**每章或每个知识块配速记口诀：**

```html
<div class="mnemonic-box">
  <h4>速记口诀</h4>
  <div class="mnemonic-item">
    <span><b>三大公式：</b>边界积分 = 内部导数积分。N-L管端点，Green管环路，Gauss管闭面。</span>
  </div>
</div>
```

**口诀设计原则**：
- 字数越少越好，最好押韵
- 只记框架，细节交给理解
- 计算型课程额外配"统一性主线"作为高阶口诀

## 知识关系图设计指南（核心创新）

### 为什么知识关系图很重要？

基础薄弱的学生最大的问题不是"记不住"，而是"不知道知识点之间有什么关系"。他们看到的是一堆孤立的概念，而不是一张有逻辑的网络。

**知识关系图的目标**：让学生一眼看到"这个知识点为什么在这个位置""它和别的知识点是什么关系"。

**注意**：计算型课程应使用"计算依赖树"替代概念关系图，展示计算层面的阻塞关系。

### 七种知识关系类型

AI在分析课程内容时，应识别知识点之间的以下关系类型，选择最合适的来组织关系图：

#### 1. 因果关系（A导致B）
**适用场景**：经济学、物理、化学等有因果链条的学科

#### 2. 前提依赖（理解A才能理解B）
**适用场景**：数学、编程等有严格知识层级的学科

#### 3. 对比关系（A vs B）
**适用场景**：任何有对立/对照概念的课程

#### 4. 递进关系（从简单到复杂）
**适用场景**：数学、编程等循序渐进的学科

#### 5. 总分关系（一个大概念包含子概念）
**适用场景**：任何有层级结构的课程

#### 6. 应用关系（理论→实际问题）
**适用场景**：理论+应用型课程

#### 7. 循环反馈（A影响B，B反过来影响A）
**适用场景**：经济学、生态学等有反馈机制的课程

#### 8. 计算阻塞（A算不对则B全废）（v2.1新增）
**适用场景**：计算型课程，展示计算层面的依赖和阻塞关系
**视觉表现**：
```
偏导数 →(计算阻塞)→ 全微分 →(计算阻塞)→ 重积分 →(计算阻塞)→ Green公式
```

### 如何选择关系类型？

AI应根据课程特点选择**1-2种主要关系类型**：

| 课程类型 | 推荐关系类型 | 示例 |
|----------|-------------|------|
| 经济学/金融 | 因果关系 + 循环反馈 | 货币政策传导机制 |
| 数学 | **计算阻塞 + 递进关系** | 微积分计算链 |
| 编程 | 前提依赖 + 总分关系 | 数据结构层级 |
| 历史 | 因果关系 + 时间线 | 事件因果链 |
| 物理 | 因果关系 + 应用关系 | 力学→运动学→能量 |
| 化学 | 总分关系 + 对比关系 | 元素周期表分类 |
| 文学 | 对比关系 + 总分关系 | 文学流派对比 |

## 代码生成避坑指南（最高优先级）

> **警告**：AI 生成 JavaScript 代码时极易引入语法错误，一个小错误会导致**整个脚本块失效**，所有功能（题库、搜索、进度保存等）全部挂掉。请严格遵守以下规则。

### 红线规则：字符串引号问题

**题库数据中所有字符串统一使用单引号包裹**，内部可以安全使用中文双引号。

```javascript
// 正确写法
{
  question: '下列哪个属于"资产"？',
  explanation: '库存现金是企业拥有的货币资金，属于"流动资产"。'
}
// 错误写法（绝对禁止！）
{
  question: "下列哪个属于"资产"？",  // 双引号嵌套，语法错误！
}
```

### KaTeX相关避坑（v2.1新增）

1. **数学模式中禁止中文**：`\text{收敛}` 会触发KaTeX警告
2. **strict:false**：渲染配置中务必加上
3. **渲染时机**：展开手风琴、切换内容后需重新调用 `renderMathInElement`
4. **$$与$配对**：确保所有公式定界符正确配对

详见 `docs/ai-pitfalls.md`。
