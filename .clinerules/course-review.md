# Course Review Skill — 基于费曼学习法的课程速通复习方案生成器

> **核心理念**：用费曼学习法，帮基础薄弱的学生在最短时间内建立课程框架、抓住考试重点。
> **不是固定模板**：AI应根据课程特点自主增减模块，本skill提供的是设计思路和参考实现，而非刻板框架。

---

## 给AI的阅读指南（重要）

如果你是AI，正在阅读这个skill来为用户创建复习方案，请遵循以下原则：

### 第一原则：费曼学习法是灵魂

你创建的每一个复习页面，都必须贯彻费曼学习法的四步精神：

| 费曼步骤 | 在复习页中的体现 | 你必须做到 |
|----------|-----------------|-----------|
| ① 用最简单的话解释 | 每个概念卡片的"白话版" | 用日常语言重新描述，禁止术语堆砌 |
| ② 用类比建立直觉 | 每个核心概念配类比 | 找一个学生熟悉的场景/模型做类比 |
| ③ 主动暴露盲区 | "常见误区" + 自测清单 | 写出学生最容易犯的错，而非只写正确答案 |
| ④ 压缩为速记口诀 | 速记口诀区 | 用最少的字把核心框架记住 |

### 第二原则：灵活组合，而非照搬模板

**不要照搬任何已有复习页的全部模块。** 你应该分析课程特点，决定：
- **哪些模块需要？** 概念型课程不需要公式速查；文科课程不需要计算题库
- **模块顺序如何排？** 编程课可能先放代码示例再放概念；历史课可能先放时间线
- **内容深度多深？** 完全零基础的学生需要更多白话和类比；有一定基础的可以更精炼
- **需要哪些题型？** 看考试形式——纯选择题的考试不需要简答题库

### 第三原则：知识关系图是核心

知识关系图不是简单的"第1章→第2章→第3章"线性罗列，而是要展示**知识点之间的逻辑关系**。详见下文"知识关系图设计指南"。

### 第四原则：面向基础薄弱+短时间

你的读者是：
- 对课程整体框架缺乏认知的人
- 核心概念理解不牢固的人
- 只有1-3天复习时间的人
- 需要直观、通俗解释而非学术语言的人

**每一个设计决策都要问自己：基础薄弱的学生看得懂吗？短时间能看完吗？**

### 第五原则：课程类型决定设计哲学（v2.1新增）

**不同类型的课程需要不同的"降维"策略。** 费曼白话法（语言降维）适合概念型/文科课程，但对计算型/空间想象型课程可能失效甚至误导。AI必须先判断课程类型，再决定采用哪种设计哲学。详见下文"计算型课程差异化设计指南"。

## 指令

### 步骤一：分析课程信息

向用户收集以下信息（如用户未提供）：

1. 课程名称
2. 课程类型（概念型/计算型/实践型/混合型）
3. 章节范围
4. 考试形式（选择题/简答题/计算题/论文/混合）
5. 学生基础（完全零基础/有一定基础）
6. 复习时间（1天/2天/3天）
7. 核心概念（3-5个最重要的概念）

### 步骤二：确定模块组合

根据课程类型选择模块：

**必选模块（任何课程都必须包含）：**

| 模块 | 费曼法对应 | 为什么必选 |
|------|-----------|-----------|
| 新手引导区 | 整体认知 | 基础薄弱学生的第一入口 |
| 知识关系图 | 理解关系 | 用关系代替背诵 |
| 概念精讲区 | 白话+类比 | 费曼法核心实践 |
| 自测清单 | 暴露盲区 | 主动发现"我不会什么" |

**可选模块（按需选用，不要全部加上）：**

| 模块 | 适用课程 | 不适用课程 |
|------|---------|-----------|
| 悬浮解释框 | 有专业术语的课程 | 概念极少的课程 |
| 概念搜索面板 | 概念多、需快速定位 | 概念少（5个以内） |
| 复习仪表盘 | 内容多、需进度管理 | 内容少（3章以内） |
| 公式速查 | 理科、计算类 | 纯文科、概念类 |
| 概念图表化 | 有图形/模型/流程 | 纯文字概念 |
| 应试急救包 | 有固定题型 | 开卷考、论文考 |
| 题库系统 | 需大量练习 | 以理解为主 |
| 章节优先级筛选 | 章节多需取舍 | 章节少 |
| 代码示例 | 编程类 | 非编程类 |
| 时间线/历史脉络 | 历史、法律 | 数学、编程 |
| 对比表格 | 有大量对比 | 概念独立 |
| 考前一页纸 | 任何课程 | — |
| **知识盲区诊断**（v2.1新增） | 任何课程 | — |
| **计算依赖树**（v2.1新增） | 计算链式递进课程 | 概念独立 |
| **统一性主线表**（v2.1新增） | 有统一性主线的课程 | 概念独立 |
| **方法决策树**（v2.1新增） | 有多种方法选择 | 只有一种方法 |
| **交互式几何可视化**（v2.1新增） | 依赖空间想象 | 纯文字概念 |
| **分步计算填空**（v2.1新增） | 计算型课程 | 纯概念型 |
| **计算陷阱库**（v2.1新增） | 计算型课程 | 纯概念型 |

**决策流程：**

```
分析课程 → 确定课程类型 → 判断是否需要差异化设计 → 选择模块 → 决定顺序 → 填充内容
    │
    ├── 概念型课程（如哲学、文学史）
    │   → 新手引导 + 知识关系图(总分/对比) + 概念精讲 + 时间线 + 自测清单
    │
    ├── 计算型课程（如高数、物理、线代）──→ 见"计算型课程差异化设计指南"
    │   → 新手引导 + 诊断测试 + 计算依赖树 + 统一性主线表 + 方法决策树
    │     + 概念精讲(几何直观版) + 几何可视化 + 分步计算填空 + 计算陷阱库
    │     + 题库 + 自测清单 + 考前一页纸
    │
    ├── 实践型课程（如编程）
    │   → 新手引导 + 知识关系图(前提依赖) + 概念精讲 + 代码示例 + 题库 + 自测清单
    │
    └── 混合型课程（如经济学）
        → 新手引导 + 知识关系图(因果/循环) + 概念精讲 + 概念图表化 + 公式速查 + 应试急救包 + 题库 + 自测清单
```

## 计算型/空间想象型课程差异化设计指南（v2.1核心新增）

> **背景**：费曼白话法（语言降维）适合概念型/文科课程，但对高数、物理、线代等强计算+强空间想象的课程，强制"生活类比"会失效甚至误导。本章节定义了这类课程的差异化设计逻辑。

### 何时启用差异化设计

当课程具备以下**任一特征**时，应在费曼白话法基础上叠加（或部分替代）差异化设计：

| 特征 | 示例课程 | 差异化设计的价值 |
|------|---------|----------------|
| 大量数学公式和符号运算 | 高数、线代、概率论 | 纯文本无法表达公式，必须KaTeX渲染 |
| 强依赖空间想象和几何直觉 | 解析几何、电磁学、力学 | 生活类比无法建立空间直觉，必须可视化 |
| 知识点是计算链式递进 | 高数（偏导→重积分→三大公式） | 前一步算不对后面全错，需要阻塞预警 |
| 存在"统一性主线" | 高数（N-L→Green→Gauss→Stokes） | 找到"纲"可以一统多，而非按章死记 |
| 有多种方法/坐标系选择 | 高数（直角/极/柱/球面坐标） | 选对方法就成功一半，需要决策引导 |

### 费曼四步法的适用性调整

计算型课程中，费曼四步法的映射关系需要调整：

| 费曼步骤 | 概念型课程（标准费曼） | 计算型课程（差异化） |
|----------|---------------------|---------------------|
| ① 用最简单的话解释 | 白话版（日常语言） | **几何直观优先**：用图形/动画建立直觉，辅以白话 |
| ② 用类比建立直觉 | 生活场景类比（吃饭、排队） | **数学类比**：低维→高维推广、特殊→一般 |
| ③ 主动暴露盲区 | 概念理解误区 | **计算操作陷阱**：忘乘r、方向弄反、未检查封闭等 |
| ④ 压缩为口诀 | 速记口诀 | 速记口诀 + **统一性主线**：找课程"纲" |

**关键原则**：
- 不强制每个概念配生活类比——对抽象数学概念，强行找生活类比是牵强甚至误导的
- 用"几何直观"替代"白话解释"作为第一直觉建立方式
- 用"低维→高维推广"替代"生活场景类比"
- 一张能动的图胜过一千字白话

### 六大设计原则

#### 原则1：公式渲染基础设施

计算型课程的公式必须用 KaTeX/MathJax 渲染，**禁用纯文本写数学公式**。这是基础设施，不可省略。

```html
<!-- 在head中引入KaTeX -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/katex.min.css">
<script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/katex.min.js"></script>
<script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/contrib/auto-render.min.js"></script>

<!-- 渲染配置 -->
<script>
function renderMath(){
  if(typeof renderMathInElement !== 'undefined'){
    renderMathInElement(document.body,{
      delimiters:[
        {left:'$$',right:'$$',display:true},
        {left:'$',right:'$',display:false}
      ],
      throwOnError:false,
      strict:false  // 容忍中文混入数学模式
    });
  }
}
</script>
```

#### 原则2：几何直观优先

**用图形和动画建立直觉，替代白话解释。** 对高数而言：
- 偏导数 → "站在山坡上，只看东西方向的坡度"（几何：切线斜率）
- 梯度 → "指向最陡上坡方向的箭头"（几何：法向量）
- 二重积分 → "曲顶柱体的体积"（几何：3D分割图）

**类比方式的转变**：
- 用"低维→高维推广"：定积分（面积）→ 二重积分（体积）→ 三重积分（超体积）
- 用"特殊→一般"：Green公式是Stokes公式的二维特例
- 用"几何意义"：旋度=旋转程度，散度=源汇强度

#### 原则3：计算依赖链+阻塞预警

计算型课程的知识关系是**计算层面的递进链**，而非概念层面的关系。学生真正需要知道的是"如果你算不对偏导数，后面全错"这种**计算阻塞关系**。

**用计算依赖树替代概念关系图**：
- 展示计算链：偏导数 → 全微分 → 梯度 → Lagrange乘数法
- 标注红色"计算阻塞点"：偏导数算不对 → 重积分全错 → 三大公式全废
- 点击节点可查看详情和跳转

#### 原则4：找统一性主线

**这是计算型课程最核心的"纲"。** 识别贯穿全书的统一主线，用一张表让学生看到"原来这些公式本质相同"。

**示例（高数下册）**：
```
Newton-Leibniz（一维：边界值 = 区间上导数的积分）
    ↓ 推广到二维
Green公式（环路积分 = 区域内偏导的积分）
    ↓ 推广到三维
Gauss公式（闭曲面积分 = 体内散度的积分）
    ↓ Green的三维版
Stokes公式（边界环量 = 曲面旋度的积分）
```

让学生看到"四个公式是一回事"，而非四个孤立公式。这比费曼的"生活类比"更能建立深层理解。

#### 原则5：分步计算填空

**用交互式推导链替代静态题库。** 将关键计算流程做成分步填空：
```
题目 → 第1步(填空) → 校验 → 第2步(填空) → 校验 → ... → 最终结果
```
每步即时反馈对错，而非只看最终答案。这比静态展示解题步骤更有效。

#### 原则6：知识盲区诊断测试

**在页面开头先做核心诊断题，自动标记盲区并跳转学习。**

流程设计：
```
打开页面 → 做6道核心诊断题（每题关联1个知识点）
    ↓
答错或点"不会" → 自动标记为盲区
    ↓
生成盲区报告 → 显示评级 + 盲区列表
    ↓
点击"跳转学习" → 展开对应章节 + 滚动到概念精讲
```

这让复习更有针对性：先测出哪里不会，再精准补漏，而不是盲目从头看到尾。

### 计算型课程专属模块详解

| 模块 | 作用 | 实现要点 | 适用条件 |
|------|------|---------|---------|
| KaTeX公式渲染层 | 数学公式排版 | CDN引入，strict:false容错 | 有数学公式 |

### 概念精讲区的差异化结构

计算型课程的概念卡片应调整结构：

```html
<div class="concept-card">
  <h3>概念名称 <span class="beginner-badge">难度标识</span></h3>

  <!-- 1. 定义（用KaTeX渲染公式） -->
  <div class="formula-box">$$公式$$</div>

  <!-- 2. 几何直观（替代白话版） -->
  <div class="geo-intuition">
    <strong>几何直观：</strong>用图形/动画描述，而非日常语言
  </div>

  <!-- 3. 数学类比（替代生活类比） -->
  <div class="plain-explain">
    <strong>低维类比：</strong>定积分是面积，二重积分是体积，维度升一级
  </div>

  <!-- 4. 应试要点 -->
  <div class="key-point">
    <strong>应试要点：</strong>考试怎么考、怎么算
  </div>

  <!-- 5. 计算陷阱（替代概念误区） -->
  <div class="common-mistake">
    <strong>计算陷阱：</strong>最容易犯的计算操作错误
  </div>
</div>
```

### KaTeX使用避坑

1. **数学模式中禁止中文**：`\text{收敛}` 会触发警告，改为 `收敛 $R=...$`
2. **strict:false**：配置中加 `strict:false` 容忍边缘情况
3. **渲染时机**：DOMContentLoaded + load 双重保险，展开手风琴后重新渲染

## 费曼学习法如何融入复习页

### 费曼法第一步：用最简单的话解释

**在概念精讲区，每个概念卡片必须包含"白话版"（概念型课程）或"几何直观"（计算型课程）：**

```html
<div class="concept-card">
  <h3>乘数效应</h3>
  <p><strong>定义：</strong>投资增加引起的国民收入增加量是投资增加量的倍数。</p>

  <!-- 概念型课程：白话版 -->
  <div class="plain-explain">
    <strong>白话版：</strong>政府花1块钱修路，修路工人拿到1块钱去买面包，
    面包师拿到钱去买衣服……这1块钱转了好几圈，最后GDP涨了不止1块钱。
    涨了几倍，这个"倍"就是乘数。
  </div>
</div>
```

**关键要求**：
- 禁止用另一个专业术语解释专业术语
- 概念型课程用日常语言；计算型课程用几何直观
- 宁可啰嗦也不要含糊

### 费曼法第二步：用类比建立直觉

**每个核心概念配一个类比：**
- **概念型课程**：生活场景类比（吃饭、排队、买东西）
- **计算型课程**：数学类比（低维→高维推广、特殊→一般）

```html
<div class="plain-explain">
  <strong>生活类比：</strong>
  IS-LM模型就像跷跷板——商品市场坐在一头，货币市场坐在另一头，
  利率是中间的支点。
</div>
```

**类比设计原则**：
- 概念型课程用学生一定熟悉的场景
- 计算型课程用数学推广关系（如"Green是Stokes的二维特例"）
- 类比要抓住概念的**核心机制**，而非表面相似

### 费曼法第三步：主动暴露盲区

**每个概念卡片包含"常见误区"（概念型）或"计算陷阱"（计算型）：**

```html
<!-- 概念型课程：概念理解误区 -->
<div class="common-mistake">
  <strong>常见误区：</strong>
  ① 很多人以为乘数越大越好——其实乘数大意味着经济波动更剧烈
</div>

<!-- 计算型课程：计算操作陷阱 -->
<div class="common-mistake">
  <strong>计算陷阱：</strong>
  ① 极坐标换元后忘记乘 r——这是最高频错误！
  ② 第二类曲线积分反向时忘记变号
</div>
```

### 费曼法第四步：压缩为速记口诀

**每章或每个知识块配速记口诀：**

```html
<div class="mnemonic-box">

  <div class="mnemonic-item">
    <span><b>三大公式：</b>边界积分 = 内部导数积分。N-L管端点，Green管环路，Gauss管闭面。</span>
  </div>

```

**口诀设计原则**：
- 字数越少越好，最好押韵
- 只记框架，细节交给理解
- 计算型课程额外配"统一性主线"作为高阶口诀

## 知识关系图设计指南（核心创新）

### 为什么知识关系图很重要？

基础薄弱的学生最大的问题不是"记不住"，而是"不知道知识点之间有什么关系"。他们看到的是一堆孤立的概念，而不是一张有逻辑的网络。

**知识关系图的目标**：让学生一眼看到"这个知识点为什么在这个位置""它和别的知识点是什么关系"。

**注意**：计算型课程应使用"计算依赖树"替代概念关系图，展示计算层面的阻塞关系。

### 七种知识关系类型

AI在分析课程内容时，应识别知识点之间的以下关系类型，选择最合适的来组织关系图：

#### 1. 因果关系（A导致B）
**适用场景**：经济学、物理、化学等有因果链条的学科

#### 2. 前提依赖（理解A才能理解B）
**适用场景**：数学、编程等有严格知识层级的学科

#### 3. 对比关系（A vs B）
**适用场景**：任何有对立/对照概念的课程

#### 4. 递进关系（从简单到复杂）
**适用场景**：数学、编程等循序渐进的学科

#### 5. 总分关系（一个大概念包含子概念）
**适用场景**：任何有层级结构的课程

#### 6. 应用关系（理论→实际问题）
**适用场景**：理论+应用型课程

#### 7. 循环反馈（A影响B，B反过来影响A）
**适用场景**：经济学、生态学等有反馈机制的课程

#### 8. 计算阻塞（A算不对则B全废）（v2.1新增）
**适用场景**：计算型课程，展示计算层面的依赖和阻塞关系
**视觉表现**：
```
偏导数 →(计算阻塞)→ 全微分 →(计算阻塞)→ 重积分 →(计算阻塞)→ Green公式
```

### 如何选择关系类型？

AI应根据课程特点选择**1-2种主要关系类型**：

| 课程类型 | 推荐关系类型 | 示例 |
|----------|-------------|------|
| 经济学/金融 | 因果关系 + 循环反馈 | 货币政策传导机制 |
| 数学 | **计算阻塞 + 递进关系** | 微积分计算链 |
| 编程 | 前提依赖 + 总分关系 | 数据结构层级 |
| 历史 | 因果关系 + 时间线 | 事件因果链 |
| 物理 | 因果关系 + 应用关系 | 力学→运动学→能量 |
| 化学 | 总分关系 + 对比关系 | 元素周期表分类 |
| 文学 | 对比关系 + 总分关系 | 文学流派对比 |

## 新手引导区设计指南

用最短时间让基础薄弱的学生建立**整体认知**。

### 必须包含的元素

- 明确标注"放弃"的章节（性价比低的）

### 白话版的写作要求

**好的白话版**：
> 乘数效应：政府花1块钱修路，修路工人拿到钱去买面包，面包师拿到钱去买衣服……这1块钱转了好几圈，最后GDP涨了不止1块钱。

**差的白话版**：
> 乘数效应：投资增加导致国民收入成倍增加的效应。

**区别**：好的白话版有**具体场景**和**动作过程**，差的白话版只是换了个说法重复定义。

## 代码生成避坑指南（最高优先级）

> **警告**：AI 生成 JavaScript 代码时极易引入语法错误，一个小错误会导致**整个脚本块失效**，所有功能（题库、搜索、进度保存等）全部挂掉。请严格遵守以下规则。

### 红线规则：字符串引号问题

**题库数据中所有字符串统一使用单引号包裹**，内部可以安全使用中文双引号。

```javascript
// 正确写法
{
  question: '下列哪个属于"资产"？',
  explanation: '库存现金是企业拥有的货币资金，属于"流动资产"。'
}
// 错误写法（绝对禁止！）
{
  question: "下列哪个属于"资产"？",  // 双引号嵌套，语法错误！
}
```

### KaTeX相关避坑（v2.1新增）

1. **数学模式中禁止中文**：`\text{收敛}` 会触发KaTeX警告
2. **strict:false**：渲染配置中务必加上
3. **渲染时机**：展开手风琴、切换内容后需重新调用 `renderMathInElement`
4. **$$与$配对**：确保所有公式定界符正确配对

详见 `docs/ai-pitfalls.md`。
