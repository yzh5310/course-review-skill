🗺️ 这章讲什么？ 就像评价一辆车要看速度和油耗一样，评价一个算法要看它快不快（时间复杂度）和费不费内存（空间复杂度）。这章介绍如何用"大O记号"来衡量算法效率。

时间复杂度

定义：算法执行时间与输入规模n的增长关系，用大O记号表示。

白话版：时间复杂度就是"当数据量变大时，算法变慢多少倍"。比如O(n)表示数据量翻倍，时间也翻倍；O(n²)表示数据量翻倍，时间变4倍。
常见复杂度排序：
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
⚠️ 常见误区：
① O(2n) = O(n)，常数系数可以省略
② O(n² + n) = O(n²)，只保留最高阶项
③ 最坏时间复杂度 vs 平均时间复杂度，考试通常考最坏

空间复杂度

定义：算法执行过程中额外占用的存储空间与输入规模n的关系。

白话版：空间复杂度就是"算法需要多少额外内存"。递归算法通常空间复杂度较高，因为每次递归都要保存现场。
🗺️ 这章讲什么？ 线性表是最基础的数据结构，就像一排座位：数组是电影院座位（连续、编号固定），链表是火车车厢（分散、靠指针连接）。

数组 vs 链表

特性数组链表
存储方式连续内存分散内存，指针连接
随机访问O(1)O(n)
插入/删除O(n)O(1)（已知位置）
空间利用率可能浪费按需分配
生活类比：
• 数组：电影院座位——找座位快（直接看编号），但中间加座位很难
• 链表：火车车厢——找某节车厢要一节节走，但中间加车厢很容易

栈与队列

栈（Stack）：后进先出（LIFO），像叠盘子。

队列（Queue）：先进先出（FIFO），像排队买票。

应用场景：
• 栈：函数调用、括号匹配、表达式求值、浏览器后退
• 队列：打印任务、消息队列、广度优先搜索（BFS）
应试要点：栈和队列的操作时间复杂度都是O(1)。考试常考：入栈出栈序列判断、用两个栈实现队列。
🗺️ 这章讲什么？ 树是一种层次结构，像公司组织架构：CEO在顶层，下面是部门经理，再下面是员工。二叉树是每个节点最多有两个子节点的树。

二叉树的遍历

四种遍历方式：

  • 前序（Pre-order）：根→左→右
  • 中序（In-order）：左→根→右
  • 后序（Post-order）：左→右→根
  • 层序（Level-order）：逐层从左到右
记忆技巧："前中后"指的是根什么时候访问。前序=根在前，中序=根在中，后序=根在后。左右顺序永远是先左后右。
应试要点：
① 给前序+中序，可以唯一确定一棵二叉树
② 给后序+中序，也可以唯一确定
③ 给前序+后序，不能唯一确定！

二叉搜索树（BST）

性质：左子树所有节点 < 根节点 < 右子树所有节点

白话版：BST就像字典——左边的词比中间小，右边的词比中间大。查找时可以"二分"，效率很高。
核心操作复杂度：
查找/插入/删除：平均O(log n)，最坏O(n)（退化成链表）
中序遍历BST会得到有序序列！
🗺️ 这章讲什么？ 图是最复杂的数据结构，像地铁线路图：站点是顶点，线路是边。图可以表示"多对多"的关系。

图的存储

邻接矩阵：用二维数组存，适合稠密图。

邻接表：用链表存，适合稀疏图。

生活类比：
• 邻接矩阵：像航班表——所有城市之间的航班都列出来，没航班的填0
• 邻接表：像每个城市的"可达城市列表"——只列出有航班的城市

图的遍历

深度优先搜索（DFS）：像走迷宫——一条路走到黑，走不通再回溯。

广度优先搜索（BFS）：像水波扩散——先把周围的都走完，再走更远的。

时间复杂度：
邻接矩阵：O(n²)
邻接表：O(n + e)（n是顶点数，e是边数）

最短路径算法

Dijkstra算法：单源最短路，不能处理负权边。

Floyd算法：多源最短路，可以处理负权边（不能有负环）。

应试要点：
Dijkstra：贪心思想，每次选最近的点，O(n²)或O((n+e)log n)
Floyd：动态规划，三重循环，O(n³)

二分查找

前提：数组有序

思路：每次取中间元素比较，缩小一半范围

生活类比：猜数字游戏——"大了"还是"小了"，每次排除一半。
时间复杂度：O(log n)
易错点：边界条件（left <= right 还是 left < right？）

哈希表

原理：通过哈希函数直接计算存储位置，O(1)查找。

冲突解决：开放定址法、链地址法

生活类比：哈希表像"存包柜"——根据你的特征（哈希函数）直接给你分配柜子号，不用一个个找。
应试要点：
① 好的哈希函数应该均匀分布
② 装填因子 = 元素个数 / 表长
③ 冲突是不可避免的，只能尽量减少

八大排序算法对比

算法平均最坏最好稳定性
冒泡排序O(n²)O(n²)O(n)稳定
选择排序O(n²)O(n²)O(n²)不稳定
插入排序O(n²)O(n²)O(n)稳定
希尔排序O(n^1.3)O(n²)O(n)不稳定
归并排序O(n log n)O(n log n)O(n log n)稳定
快速排序O(n log n)O(n²)O(n log n)不稳定
堆排序O(n log n)O(n log n)O(n log n)不稳定
基数排序O(d(n+r))O(d(n+r))O(d(n+r))稳定
记忆口诀：
"快选堆希不稳定"（快速、选择、堆、希尔是不稳定的）
"归并插冒泡稳定"（归并、插入、冒泡是稳定的）

快速排序

思想：分治法——选一个基准，比它小的放左边，比它大的放右边，递归处理。

生活类比：像老师按身高排队——先找一个同学当基准，比他矮的站左边，比他高的站右边，然后左右两边再各自找基准继续排。
应试要点：
① 平均O(n log n)，最坏O(n²)（已有序时）
② 不稳定排序
③ 实际应用中最快的排序算法
